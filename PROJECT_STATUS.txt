                              KEY CHALLENGES
═══════════════════════════════════════════════════════════════════════════════

🔥 HIGH SEVERITY
   1. API Rate Limiting (4+ providers with different limits)
      └─ Finnhub: 60/min | CoinGecko: 50/min, 10k/month | FMP: 250/day

   2. yfinance Reliability (unofficial API, breaks when Yahoo changes)
      └─ Solution: Use as fallback only, implement robust error handling

   3. WebSocket Connection Management
      └─ Finnhub: 1 connection per API key
      └─ Must multiplex to multiple frontend clients

⚠️  MEDIUM SEVERITY
   4. Data Model Transformation
      └─ Each API returns different JSON structure
      └─ Must transform to unified Quote/OHLCV models

   5. Crypto vs Stock Symbol Resolution
      └─ "BTC" could be Bitcoin or Bolivian Time Corp
      └─ Need asset_type parameter or smart detection
